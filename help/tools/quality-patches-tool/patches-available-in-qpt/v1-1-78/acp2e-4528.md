# ACP2E-4528: Customer address city validation mishandles forward slashes and invalid characters such as !, ", #, and ?

The ACP2E-4528 patch fixes the issue where customer address city validation incorrectly allowed invalid characters such as !, ", #, and ?, while properly supporting the forward slash (/). This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4528. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

City validation in customer addresses incorrectly allows invalid characters such as !, ", #, and ?, while handling forward slash (/) incorrectly, allowing invalid city names to be saved.

<u>Steps to reproduce</u>:

1. Log in to the Storefront as a registered customer.
1. Navigate to **[!UICONTROL My Account]** > **[!UICONTROL Address Book]**.
1. Click **[!UICONTROL Add New Address]** or edit an existing address.
1. In the **[!UICONTROL City]** field, enter *PARGAS-5xx!"#?!x*.
1. Save the address.

<u>Expected results</u>:

A validation error should appear: `Invalid City. Please use letters, numbers, spaces, and the following characters: - _ ' ’ . , & ( )  /.`

<u>Actual results</u>:

It is possible to save a city name containing invalid characters such as !, ", #, and ?.
