# ACP2E-4555: Telephone numbers that contain periods or forward slashes don't validate properly

The ACP2E-4555 patch fixes the issue where modern telephone numbers containing `.` or `/` aren't validated properly. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4555. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Modern telephone numbers containing `.` or `/` aren't validated properly.

<u>Steps to reproduce</u>:

1. Log in as a customer on the Storefront.
1. Add a new address to the address book with a telephone number, for example: `06.76.40.32.22` or `+43680/2149568`.
1. Save the address details.

<u>Expected results</u>:

It's possible to use modern telephone formats with periods and forward slashes.

<u>Actual results</u>:

Validation fails for the telephone field due to the `.` or `/` characters.
