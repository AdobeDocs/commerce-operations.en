# ACP2E-4452: Quick Order page product prices include tax regardless of tax display configuration

The ACP2E-4452 patch fixes the issue where product prices on the Quick Order page include tax regardless of the tax display configuration. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4452. Please note that this issue is scheduled to be fixed in Adobe Commerce B2B 1.5.2-p3.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) B2B 1.5.2-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) B2B >=1.5.1 <1.5.3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Product prices on the Quick Order page include tax regardless of the tax display configuration.

<u>Prerequisites</u>:

1. [!DNL B2B] must be installed on the Adobe Commerce instance.

<u>Steps to reproduce</u>:

1. Enable the [!DNL B2B] Quick Order feature:
    1. Go to **Stores** > **Settings** > **Configuration** > **[!DNL B2B] Features**.
    1. Set **Enable Quick Order** to **Yes**.
1. Configure taxes:
    1. Create a tax zone named Global in [!UICONTROL Stores] > [!UICONTROL Taxes] > [!UICONTROL Tax Zones and Rates] for the required country, covering all states/regions and ZIP codes, with a tax rate of 20%.
    1. Change one setting in **Stores** > **Settings** > **Configuration** > **Sales** > **Tax** > **Shopping Cart Display Settings** > **Display Prices** = **Including and Excluding Tax**.
1. Create a simple product with SKU = Test and a price = $10.00.
1. Create a customer account with the US postcode, e.g., 90210.
1. Go to the product page and verify that prices display without tax.
1. Go to the **Quick Order** page.
1. Find the product by SKU = Test.

<u>Expected results</u>:

The price should match the product page price, excluding tax.

<u>Actual results</u>:

The displayed price of $12.00 has tax applied.
