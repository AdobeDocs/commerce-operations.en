# ACP2E-4628: Importing customers with uppercase email addresses triggers an undefined array key error when Account Sharing is set to Global

The ACP2E-4628 patch fixes the issue where importing customers with uppercase email addresses causes an undefined array key error when **Account Sharing** is set to **Global**, resulting in import failure. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4628. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p1

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Customer import fails with an undefined array key error when uppercase email addresses are used with Global Account Sharing enabled.

<u>Steps to reproduce</u>:

1. Go to **[!UICONTROL Stores]** > **Settings** > **Configuration** > **[!UICONTROL Customers]** > **[!UICONTROL Customer Configuration]** > **[!UICONTROL Account Sharing Options]**.
    1. Set **[!UICONTROL Share Customer Accounts]** to *Global*.
1. Go to **[!UICONTROL System]** > **[!UICONTROL Export]** > **[!UICONTROL Customer Main File]** and export an existing customer.
1. Go to **[!UICONTROL System]** > **[!UICONTROL Import]** > **[!UICONTROL Customer Main File]**.
    1. Set [!UICONTROL Import Behavior] to Add/Update Complex Data.
    1. Upload the exported file without making any changes.
1. Click the **[!UICONTROL Check Data]** button.

<u>Expected results</u>:

The customer import should complete successfully without any errors, even when email addresses contain uppercase characters and Account Sharing is set to Global.

<u>Actual results</u>:

The import fails with the following error:

```
Warning: Undefined array key "Uppercase.Email@example.com" in app/code/Magento/CustomerImportExport/Model/ResourceModel/Import/Customer/Storage.php on line 117
```
