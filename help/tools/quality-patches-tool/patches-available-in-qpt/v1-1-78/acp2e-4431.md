# ACP2E-4431: [!UICONTROL Related Products] matched by target rules are deleted during the reindex process

The ACP2E-4431 patch fixes the issue where the **[!UICONTROL Related Products]** matched by the target rules are deleted during the reindex process. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4431. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p4

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

The **[!UICONTROL Related Products]** that are matched by the target rules are deleted during the reindex process.

<u>Steps to reproduce</u>:

1. Create 10000 or more products.
1. Assign 10000 or more products to 5000 or more categories.
1. Create 100 or more related product rules.
1. Configure rule conditions where **[!UICONTROL Products]** to match use **[!UICONTROL Product Category]** is *X*.
1. Configure **[!UICONTROL Products]** to display using **[!UICONTROL Product Category]** is **[!UICONTROL Constant Value]** *Y*.
1. Execute this GraphQL query to retrieve `related_products` for a product:

    ```graphql
    {
        products(filter: { url_key: { eq: "simple-product-491" } }) {
            items {
                sku
                related_products {
                    uid
                    sku
                    name
                    url_key
                    stock_status
                    price_range {
                        minimum_price {
                            regular_price {
                               value
                            }
                            final_price {
                                value
                            }
                        }
                    }
                }
            }
        }
    }
    ```

1. Create a large backlog for the `targetrule_product_rule` index.
1. Create a large backlog for the `targetrule_rule_product` index.
1. Run the `targetrule_rule_product` indexer.
1. Run the `targetrule_product_rule` indexer.

<u>Expected results</u>:

The `related_products` list is always populated with products that match the rule conditions.

<u>Actual results</u>:

The `related_products` list returns an empty array while the indexers are executing.
