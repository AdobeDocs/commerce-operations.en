# ACP2E-4535: Submitting the forgot-password form destroys or regenerates the session (PHPSESSID changes) and clears the guest cart

The ACP2E-4535 patch fixes the issue where submitting the forgot-password form causes the session to be destroyed or regenerated (`PHPSESSID` changes) and clears the guest cart. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4535. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p1

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where submitting the forgot-password form destroys or regenerates the session (PHPSESSID changes) and clears the guest cart.

<u>Steps to reproduce</u>:

1. Add a product to the cart as a guest.
1. Verify that the mini-cart displays one item.
1. Open the browser’s developer tools and navigate to **[!UICONTROL Application]** > **[!UICONTROL Cookies]**. Note the current PHPSESSID value.
1. On the storefront, navigate to `/customer/account/forgotpassword/`.
1. Enter any email address in the email field.
1. Complete the CAPTCHA validation.
1. Click **[!UICONTROL Reset My Password]**.
1. Wait for the success message to appear.
1. Check the `PHPSESSID` cookie value again.
1. Review the mini-cart contents.

<u>Expected results</u>:

The cart retains all items.

<u>Actual results</u>:

After submitting the Forgot Password form, the `PHPSESSID` cookie value changes, causing the mini-cart to become empty. As a result, the guest quote is orphaned because it was associated with the previous session ID.
