# ACP2E-4419: Gift cards do not apply correctly at checkout after successful reCAPTCHA v2 validation on the storefront

The ACP2E-4419 patch fixes the issue where gift cards are not applied correctly at checkout after successful reCAPTCHA v2 ('I am not a robot') validation on the storefront. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 is installed. The patch ID is ACP2E-4419. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.6 - 2.4.7-p9

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where gift cards are not applied correctly at checkout after successful reCAPTCHA v2 ('I am not a robot') validation on the storefront.

<u>Steps to reproduce</u>:

1. Setup Google reCAPTCHA v2 ("I am not a robot") in Admin ->  Stores -> Settings -> Configuration -> Security -> Google reCAPTCHA Storefront.
1. Configure reCAPTCHA for gift cards.
1. Create a gift card account with code in Admin -> Marketing -> Gift Card Accounts.
1. On the Storefront, add a product to the cart.
1. Try to apply the gift card code to the cart.
1. Proceed to the checkout payment step and try to apply the gift card code.

<u>Expected results</u>:

1. The gift card should be successfully applied after Google reCAPTCHA validation is successful.

<u>Actual results</u>:

1. Error from reCAPTCHA:

  ```
  Unknown macro: \{"message"}
  ```
