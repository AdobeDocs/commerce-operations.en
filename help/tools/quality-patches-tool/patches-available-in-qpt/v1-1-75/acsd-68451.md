# ACSD-68451: Company admin links to unrelated company across multiple websites

The ACSD-68451 patch fixes the issue where a company admin logs in to one website, creates an unrelated company on another website, and is incorrectly associated with that unrelated company. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 is installed. The patch ID is ACSD-68451. Please note that this issue is scheduled to be fixed in Adobe Commerce [!DNL B2B] 1.5.3.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) - [!DNL B2B] 1.5.2-p2

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) - [!DNL B2B] 1.5.2-p1 - 1.5.2-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

In a multiâ€‘website setup, a company admin logs in to one website and creates a company. Then, in the same browser, the admin opens another website and creates a different company. The platform incorrectly associates the second company admin with the first company, even though the companies remain unrelated.

<u>Prerequisites</u>:

Install [!DNL B2B] and Inventory.

<u>Steps to reproduce</u>:

1. Create an additional website, store, and store view.
1. Enable *[!UICONTROL Company]* under **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**.
1. Access the default website on the storefront.
1. Create a new company on the storefront and sign in as the company administrator.
1. In the same browser session, access the secondary website.
1. Create a new company on the storefront for the secondary website.
1. Log in to the Admin panel and verify the customer grid.

<u>Expected results</u>:

The new company admin remains assigned only to the newly created company, even when another company admin stays logged in on a different website in the same browser.

<u>Actual results</u>:

The new company admin also gets assigned to the other company as a **Company User**, in addition to the intended company.
