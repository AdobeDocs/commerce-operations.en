# ACSD-68359: Fixes *414* error when selecting **[!UICONTROL Pick in Store]** with large carts

The ACSD-68359 patch fixes the issue where **[!UICONTROL Pick in Store]** fails at checkout with a *414* error when the cart contains many products. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 is installed. The patch ID is ACSD-68359. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Issue

Selecting **[!UICONTROL Pick in Store]** fails with a *414* error when the cart contains many products due to long URLs generated by the pickup location search.

**Prerequisites:**
1. Install **[!UICONTROL B2B]**.
2. Go to **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL B2B Features]** > **[!UICONTROL General Settings]** and enable **[!UICONTROL Company]** and **[!UICONTROL Quick Order]**.

<u>Steps to reproduce</u>:

1. Add 100+ products to the cart using **[!UICONTROL Quick Order]**.
1. Proceed to checkout and select **[!UICONTROL Pick in Store]**.
1. Observe the *414* error returned by the **[!UICONTROL Search for pickup location]** endpoint.

<u>Expected results</u>:

The **[!UICONTROL Search for pickup location]** endpoint functions correctly regardless of the number of products in the cart.

<u>Actual results</u>:

The **[!UICONTROL Search for pickup location]** endpoint returns a 414 error when the cart contains around 100 products.
