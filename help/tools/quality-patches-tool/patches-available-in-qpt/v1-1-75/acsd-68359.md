# ACSD-68359: Selecting a store during checkout using "Pick in Store" no longer fails due to long URLs when many products are in the cart. Previously, this triggered a 414 error caused by excessively long URLs generated during store sel

The ACSD-68359 patch fixes the issue where the "Search for pickup location" endpoint returns a 414 error when adding 100+ products to the cart and selecting the "Pick in Store" option. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 is installed. The patch ID is ACSD-68359. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

* Adobe Commerce (all deployment methods) 2.4.8-p2

## Issue

Issue where the "Search for pickup location" endpoint returns a 414 error when adding 100+ products to the cart and selecting the "Pick in Store" option.

<u>Steps to reproduce</u>:

1. Add 100 products to the cart via Quick Order.
1. Select the "Pick in Store" option at checkout.
1. Observe the 414 error returned by the "Search for pickup location" endpoint.

<u>Expected results</u>:

The "Search for pickup location" endpoint should function correctly regardless of the number of products in the cart.

<u>Actual results</u>:

The "Search for pickup location" endpoint returns a 414 error when the cart contains around 100 products.
