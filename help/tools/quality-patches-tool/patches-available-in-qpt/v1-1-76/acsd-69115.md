# ACSD-69115: Shopping cart errors not displayed to admin users for customers assigned to non‑default websites

The ACSD-69115 patch fixes the issue where shopping cart errors were not displayed to the admin user when managing the shopping cart for a customer assigned to a non-default website. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 is installed. The patch ID is ACSD-69115. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where shopping cart errors were not displayed to the admin user when managing the shopping cart for a customer assigned to a non-default website.

<u>Steps to reproduce</u>:

1. Log in to the Admin panel.
1. Update configuration to the following values:
    1. Share Customer Accounts - Per Website.
    1. Catalog Price Scope - Website.
    1. Set all indexers to 'On schedule'.
1. Update the website structure to the following:
    1. base / main_website_store / default (must be set as the default website).
    1. website_2 / website_2 / website 2.
1. Create a new customer from the Admin panel:
    1. Associate the customer with Website - website_2.
1. Create the following configurable products:
    1. conf_0:
        1. color - red.
        1. Product in Website - default, website_2.
    1. conf_1:
        1. color - red.
        1. Product In Website - website_2.
1. Run the scheduled cron jobs.
1. Edit 'website_2' customer and 'Manage Shopping Cart'.
1. 'website_2' should be selected by default.
1. Expand "Add to Shopping Cart by SKU" and add both products:
    1. conf_0, qty 1.
    1. conf_1, qty 1.
1. Click 'Add Selections to Cart' (try adding separately, can't add 'global' conf_0 and 'website_2' conf_1).

<u>Expected results</u>:

It's possible to add products through Manage Shopping Cart while on a non-default scope (website_2) - additional configuration is shown.

<u>Actual results</u>:

It isn't possible to add products through Manage Shopping Cart while on a non-default scope (website_2); configuration messages are not visible.
