# ACSD-69541: Reducing a product's quantity in the admin to less than what already exists in a cart made it impossible to edit the product quantity in that cart via GraphQL

The ACSD-69541 patch fixes the issue where adding a product to the shopping cart, and reducing its quantity in the Admin Panel makes it impossible to edit this product quantity in the same shopping cart via GraphQL later. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 is installed. The patch ID is ACSD-69541. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p2

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where adding a product to the shopping cart, and reducing its quantity in the Admin Panel makes it impossible to edit this product quantity in the same shopping cart via GraphQL later.

<u>Steps to reproduce</u>:

1. Create a simple product: SKU=p1 with quantity=10.
1. Reindex to ensure salable quantity = 10 exists.
1. Add qty = 10 of the previously created product to the shopping cart by GraphQL request.
1. Edit the product in the Admin Panel to reduce the product qty to 5 and save it.
1. Do a reindex.
1. Try to update the product qty in the shopping cart to <= 5 by GraphQL request.
1. Try to update the product qty in the shopping cart to > 5 by GraphQL request.

<u>Expected results</u>:

Cart quantity should be updated if sufficient quantity is In Stock. Otherwise corresponding error message should be returned in the GraphQL response.

<u>Actual results</u>:

When trying to update cart qty to <= 5, success response from GraphQL is received. But the product quantity in the shopping cart remains unchanged.
When trying to update cart qty to > 5, an internal server error is received in the GraphQL response instead of valid error message.
