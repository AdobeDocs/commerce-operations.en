# ACSD-67091: Catalog rule product index cleanup fails due to maximum writeset size during large deletions

The ACSD-67091 patch fixes the issue where catalog rule product index cleanup fails due to maximum writeset size during large deletions. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 is installed. The patch ID is ACSD-67091. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Catalog rule product index cleanup fails with a maximum writeset size error during large deletions.

<u>Prerequisites</u>:

Your instance must have multiple websites, more than 100,000 products, several customer groups, and more than 20 catalog price rules.

<u>Steps to reproduce</u>:

1. Run reindexing to generate data and populate the `catalogrule_product` table.
1. Make multiple catalog changes, and then run reindexing again.
1. Observe that the `catalogrule_product` table is cleared using a single delete operation.


<u>Expected results</u>:

1. Large `catalogrule_product` tables are cleared without issues.
1. The indexer deletes data in manageable batches to avoid transaction limits.
1. No critical performance degradation or failures occur during standard index operations.


<u>Actual results</u>:

1. The `catalogrule_product` table is cleared using a single large delete query that affects more than 50 million records.
1. MySQL returns error: *WSREP: transaction size limit (2147483647) exceeded.*
1. The indexer operation fails due to writeset size limitations.
1. Database performance degrades because millions of rows are deleted in a single transaction.
