# ACSD-69331: Content creators in the media gallery could not create folders with only the `create_folder` permission. After the fix, they can create folders as expected

The ACSD-69331 patch fixes the issue where content creators in the media gallery could not create folders with only the `create_folder` permission. After the fix, they can create folders as expected. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 is installed. The patch ID is ACSD-69331. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where content creators in the media gallery could not create folders with only the `create_folder` permission. After the fix, they can create folders as expected.

<u>Steps to reproduce</u>:

1. Log in to Admin Panel.
1. Create a new role with access to Dashboard and Content > Media > Media Gallery, granting only the “Create Folder” permission (no “Delete Folder” permission).
1. Create a new admin user and assign this role to them.
1. Log in as new admin user.
1. Go to Content > Media > Media Gallery.
1. Select a directory (e.g., catalog).
1. Click Create Folder.
1. Confirm a folder name.

<u>Expected results</u>:

Folder creation should work with only “Create folder” permission.
“Delete folder” permission should not be required to create the folder.

<u>Actual results</u>:

User cannot create folders with “Create folder” permission only. Additionally, "Delete Folder" permission must be granted to this role to be able to create folders.
