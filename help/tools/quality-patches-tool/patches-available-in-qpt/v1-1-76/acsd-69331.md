# ACSD-69331: Media Gallery folder creation fails with only **[!UICONTROL Create Folder]** permission

The ACSD-69331 patch fixes the issue where content creators could not create folders in the CMS Media Gallery when only the **[!UICONTROL Create Folder]** permission was granted. After the fix, they can create folders as expected. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 is installed. The patch ID is ACSD-69331. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Admin users with only the **[!UICONTROL Create Folder]** permission in the Media Gallery cannot create folders unless the **[!UICONTROL Delete Folder]** permission is also granted.

<u>Steps to reproduce</u>:

1. Log in to the Admin panel.
1. Create a new role with access to **[!UICONTROL Dashboard]** and **[!UICONTROL Content]** > **[!UICONTROL Elements]** > **[!UICONTROL Media Gallery]**, granting only the **[!UICONTROL Create Folder]** permission (do not grant **[!UICONTROL Delete Folder]**).
1. Create a new admin user and assign this role.
1. Log in as the new admin user.
1. Go to **[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**.
1. Select a directory (for example, `catalog`).
1. Click **[!UICONTROL Create Folder]**.
1. Enter and confirm a folder name.

<u>Expected results</u>:

* Folder creation works with only the **Create Folder** permission.
* The **Delete Folder** permission is not required to create folders.

<u>Actual results</u>:

* The user cannot create folders with only the **Create Folder** permission.
* Granting the **Delete Folder** permission is required to create folders.
