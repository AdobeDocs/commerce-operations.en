# ACP2E-4468: Website-scoped Admin users can't save Dynamic Blocks in Page Builder

The ACP2E-4468 patch fixes the issue where Admin users with website scope permissions can't edit banner content and receive a permissions error when saving [!UICONTROL Dynamic Blocks]. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81 is installed. The patch ID is ACP2E-4468. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Admin users with website scope permissions can't edit banner content and receive a permissions error when saving [!UICONTROL Dynamic Blocks].

<u>Prerequisites</u>:

Page Builder is enabled.

<u>Steps to reproduce</u>:

1. Go to **[!UICONTROL Stores]** > **[!UICONTROL Settings]** > **[!UICONTROL All Stores]** and create a new website, store, and store view.
1. Go to **[!UICONTROL Content]** > **[!UICONTROL Elements]** > **[!UICONTROL Dynamic Blocks]** and create a new [!UICONTROL Dynamic Block] with Page Builder content.
1. Go to **[!UICONTROL System]** > **[!UICONTROL Permissions]** > **[!UICONTROL User Roles]** and create a new role.
1. Under **[!UICONTROL Role Scopes]**, set **[!UICONTROL Role Scopes]** to *Custom*, and select the website and store view created in step 1.
1. Under **[!UICONTROL Role Resources]**, set **[!UICONTROL Resource Access]** to *All*.
1. Go to **[!UICONTROL System]** > **[!UICONTROL Permissions]** > **[!UICONTROL All Users]** and create a new Admin user assigned to the restricted role.
1. Sign in as the restricted Admin user.
1. Go to **[!UICONTROL Content]** > **[!UICONTROL Elements]** > **[!UICONTROL Dynamic Blocks]** and open the created [!UICONTROL Dynamic Block] for editing.
1. Attempt to save the changes.

<u>Expected results</u>:

An Admin user with permissions restricted to a specific website and store view can edit and save the [!UICONTROL Dynamic Block].

<u>Actual results</u>:

1. The system returns the error: *More permissions are needed to save this item.*
1. The [!UICONTROL Dynamic Block] changes aren't saved.
