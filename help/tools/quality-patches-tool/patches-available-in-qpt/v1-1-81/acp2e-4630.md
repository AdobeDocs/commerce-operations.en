# ACP2E-4630: Long product names overlap adjacent columns in multi-page sales PDFs after page breaks

The ACP2E-4630 patch fixes the issue where long product names with custom options overlap adjacent columns or items in multi-page Invoice, Shipment, Credit Memo, and Return PDFs after page breaks, making line items unreadable. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81 is installed. The patch ID is ACP2E-4630. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where long product names with custom options overlap adjacent columns or items in multi-page Invoice, Shipment, Credit Memo, and Return PDFs after page breaks, making line items unreadable.

<u>Steps to reproduce</u>:

1. Place a few orders from the storefront with 20 or more items, including products with long names and custom options.
1. In the Admin sidebar, go to Sales > Orders. Open an order and create the related invoice, shipment, credit memo, and return.
1. Generate the invoice, shipment, credit memo, and return PDFs from their respective pages.
1. Open the generated PDFs and review all pages.

<u>Expected results</u>:

1. No text overlaps in the PDF documents.
1. Product names, SKUs, quantities, prices, and custom options are displayed clearly across all pages.

<u>Actual results</u>:

1. Product names and custom options overlap with the SKU, quantity, and price columns on pages after the first page.
1. Line items become unreadable because the row height is calculated incorrectly after page breaks.
