# ACP2E-4786: Product export fails when AWS S3 remote storage is configured

The ACP2E-4786 patch fixes the issue where product export fails because the storage driver uses an invalid path when AWS S3 remote storage is enabled. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81 is installed. The patch ID is ACP2E-4786. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p4

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.9

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Product export fails when AWS S3 remote storage is enabled because of invalid path handling in the storage driver. 

<u>Steps to reproduce</u>:

1. Configure [!DNL AWS S3] as remote storage:

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. Disable database storage for media files by running the following command:

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. In the Admin sidebar, go to **[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **Export**.
1. For **[!UICONTROL Entity Type]**, choose **[!UICONTROL Products]**.
1. Click **[!UICONTROL Continue]**.

<u>Expected results</u>:

The export file is created successfully.

<u>Actual results</u>:

1. The export file is not created.
1. The logs contain a `SignatureDoesNotMatch` error.
