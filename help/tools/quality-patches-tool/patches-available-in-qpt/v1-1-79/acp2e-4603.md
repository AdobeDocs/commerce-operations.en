# ACP2E-4603: Catalog Permissions product reindex leave stale permission index rows and MySQL deadlock

The ACP2E-4603 patch fixes the issue where running the Catalog Permissions product reindex left existing permission index rows unchanged, causing updated category permission grants to not be reliably reflected on products. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 is installed. The patch ID is ACP2E-4603. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.6 - 2.4.7-p9

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where running the Catalog Permissions product reindex left existing permission index rows unchanged, causing updated category permission grants to not be reliably reflected on products.

<u>Steps to reproduce</u>:

1. Enable Category Permissions:
    1. Log in to Admin.
    1. Go to `Stores → Configuration`.
    1. Under `Catalog`, open `Catalog`.
    1. Expand `Category Permissions`.
    1. Set `Enable` to `Yes`.
    1. Save the configuration.
1. Create 50 or more customer groups (Option A or Option B):
    1. **Option A** — Admin: `Customers → Customer Groups → Add New`; set Group Name (e.g. `ACSD70034_01`) and Tax Class default; save. Repeat 50+ times.
    1. **Option B (recommended)** — REST API: call `POST /rest/V1/customerGroups` with body `{"group":{"code":"ACSD70034_01","taxClassId":0}}`, repeated for 50 groups (`ACSD70034_01` through `ACSD70034_50`).
1. Add category permissions in Admin:
    1. Go to `Catalog → Categories`.
    1. Edit 10–20 or more categories (more categories = better reproduction).
    1. In each category, scroll to `Category Permissions` and click `Add Permission`.
    1. Select a Website (e.g. `Main Website`) and one of the customer groups created above.
    1. Set `Grant Catalog Category View`, `Grant Product Price`, and `Grant Checkout` as needed.
    1. Save the category.
    1. **Recommended scale:** 20 categories × 20–50 customer groups = 400–1,000 permission rows. More rows increase the likelihood of deadlock.
1. Ensure a single index table (default):
    1. Go to `Stores → Configuration → Catalog → Catalog → Category Permissions`.
    1. If a `Dimensions` or `Indexer Dimensions` option is present, set it to `None` (not "Customer Group"). Default is a single table.
1. Run reindex with parallel threads from the server CLI:
    1. Export `MAGE_INDEXER_THREADS_COUNT=4`.
    1. Run `php bin/magento indexer:reset catalogpermissions_category catalogpermissions_product`.
    1. Run `php bin/magento indexer:reindex catalogpermissions_category catalogpermissions_product`.
    1. Repeat the reset/reindex pair 5–10 times to surface the deadlock (parallel execution is required to trigger it).

<u>Expected results</u>:

Reindex completes successfully with parallel threads, and updated category permission grants are reliably reflected on products in the `magento_catalogpermissions_index_product` table.

<u>Actual results</u>:

MySQL deadlock (error 1213) occurs intermittently during reindex; child processes fail with errors referencing `REPLACE INTO` on `magento_catalogpermissions_index_replica` or `magento_catalogpermissions_index_product`.
Even when reindex completes without an exception, existing permission index rows are not reliably updated, leaving stale permission grants on some products.
