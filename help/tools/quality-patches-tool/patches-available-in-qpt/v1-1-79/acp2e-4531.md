# ACP2E-4531: CMS hierarchy doesn't update hierarchical URL when child page URL key is changed

The ACP2E-4531 patch fixes the issue where changing a CMS page's URL key didn't update the page's hierarchical URL. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 is installed. The patch ID is ACP2E-4531. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where changing a CMS page's URL key didn't update the page's hierarchical URL.

<u>Steps to reproduce</u>:

1. Create a CMS page (for example, "help").
1. Create a second CMS page (for example, "my-account"), selecting "help" as its parent in the hierarchy.
1. Load both pages from the Storefront; both should resolve correctly.
1. Change the my-account page's URL Key (under "Search Engine Optimization") to "my-account-2".
1. Load `/my-account-2` and `/help/my-account-2` from the Storefront.

<u>Expected results</u>:

When the CMS page URL Key changes, the hierarchy URLs are updated accordingly. `/help/my-account-2` resolves to the page, and `/my-account` no longer resolves (no such URL exists).

<u>Actual results</u>:

`/my-account-2` redirects to {{{}/my-account{}}}, and `/help/my-account-2` returns a 404. The hierarchical URL is not updated when the child page URL key changes.
