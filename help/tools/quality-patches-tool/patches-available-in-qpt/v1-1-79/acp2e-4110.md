# ACP2E-4110: Bundle product prices with a **[!UICONTROL Special Price]** differ between PDP/PLP and cart in a non-default currency

The ACP2E-4110 patch fixes the issue where bundle products with a **[!UICONTROL Special Price]** display incorrect amounts on the [!UICONTROL Product Detail Page] (PDP) and the [!UICONTROL Product Listing Page] (PLP) in a non-default currency. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 is installed. The patch ID is ACP2E-4110. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Bundle products with a **[!UICONTROL Special Price]** display incorrect amounts on the PDP and the PLP in a non-default currency.

<u>Steps to reproduce</u>:

1. Create a simple product. Set the **[!UICONTROL Price]** = *$40.25*.
1. Create a bundle product.
1. Set the **[!UICONTROL Special Price]** for the bundle product = *97.5%* and the **[!UICONTROL Price View]** = *Price Range*.
1. Assign the simple product as an option of the bundle product. Set the **[!UICONTROL default quantity]** = *8*.
1. Assign the bundle product to a visible category so it can be checked on the PLP/category page.
1. Keep the pricing source in *USD*, and set the **[!UICONTROL allowed currency]** = *GBP* and set the default **[!UICONTROL display currency]** = *GBP* for the storefront.
1. Set the **[!UICONTROL currency conversion rate]** for USD to GBP = *0.75*.
1. Clear the cache and run reindex.
1. Open the storefront in the GBP view and note the bundle price on the PLP/category page and the PDP.
1. Add the bundle product to the cart.
1. Compare the displayed price in the cart with the price on the PLP/PDP.

<u>Expected results</u>:

The bundle product price should be the same on the PLP, PDP, and cart in the non-default currency.

<u>Actual results</u>:

The displayed price differs between the product page/listing and the cart.
Example:
Price on the product page/PDP/PLP = *235.52 GBP*
Price in the cart = *235.44 GBP*
