# ACP2E-4505: Circular category dependency from saving a category with stale data in a duplicate browser tab

The ACP2E-4505 patch fixes the issue where it was possible to save a category with stale data from a duplicate browser tab, creating a circular dependency. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 is installed. The patch ID is ACP2E-4505. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3, 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

It was possible to save a category with stale data from a duplicate browser tab, creating a circular dependency.

<u>Steps to reproduce</u>:

1. Create a category structure: A > B > C.
1. In the Admin panel, go to **[!UICONTROL Catalog]** > **[!UICONTROL Categories]**.
1. In the category tree, select category C to open its edit form in Tab 1.
1. Duplicate the browser tab to open Tab 2 with the same category C edit form.
1. In Tab 1:
    1. Drag category C under A so C becomes a child of A.
    1. Drag category B under C so B becomes a child of C.
    1. Wait until both drag-and-drop operations finish and the category tree updates.
1. In Tab 2 (still showing the old hierarchy):
    1. Make any change, for example, edit the description.
    1. Click **[!UICONTROL Save]**.

<u>Expected results</u>:

The system should validate and prevent circular parent-child relationships from being saved.

<u>Actual results</u>:

A circular parent-child dependency is created between categories (category B has parent C, and category C has parent B), causing infinite recursion in category hierarchy traversal.
