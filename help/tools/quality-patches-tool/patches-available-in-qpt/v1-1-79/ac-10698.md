# AC-10698: Transaction prices and totals aren't sent to [!DNL Google Tag]

The AC-10698 patch fixes the issue where the system sent the currency at the all-orders level instead of associating it with individual orders. Transaction prices and totals are now sent per order to [!DNL Google Tag], improving ecommerce data tracking accuracy. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 is installed. The patch ID is AC-10698. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.8.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.7 - 2.4.7-p9

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

The system sent the currency at the all-orders level instead of associating it with individual orders.

<u>Steps to reproduce</u>:

1. Log in to the Admin of your Commerce store.
1. On the _Admin_ sidebar, go to **[!UICONTROL Stores]** > _[!UICONTROL Settings]_ > **[!UICONTROL Configuration]**.
1. In the left panel, expand **[!UICONTROL Sales]** and choose **[!UICONTROL Google API]**.
1. Expand **[!UICONTROL Google Analytics]** and:
    * Enable the integration.
    * Select **[!UICONTROL Google Tag Manager]** as the account type.
    * Enter your GTM container ID (`GTM-xxxxxx`).
    * (Optional) Enable content experiments.
    * Leave remaining fields at their default values.
1. Add a product to the cart and complete an order through checkout.
1. On the success page, inspect the `OrderTrackingData` object that is sent to [!DNL Google Tag].

<u>Expected results</u>:

The currency is sent as part of each individual order, not at the top-level all-orders object. Transaction prices and totals are correctly associated with each order, matching the [Google Analytics 4 ecommerce documentation](https://developers.google.com/analytics/devguides/collection/ga4/ecommerce?client_type=gtag#make_a_purchase_or_issue_a_refund) expectation.

<u>Actual results</u>:

The currency is sent at the all-orders level rather than per order, so transaction prices and totals are not correctly associated with individual orders in [!DNL Google Tag] / Google Analytics ecommerce tracking.
