# ACP2E-4706: **[!UICONTROL Target Rule]** indexer cron doesn't pick products enabled only at store view scope

The ACP2E-4706 patch fixes the issue where products not enabled in the **[!UICONTROL Admin]** scope are skipped by the **[!UICONTROL Target Rule]** indexer. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 is installed. The patch ID is ACP2E-4706. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.8-p4

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Products not enabled in the **[!UICONTROL Admin]** scope are skipped by the **[!UICONTROL Target Rule]** indexer.

<u>Steps to reproduce</u>:

1. Create a simple **[!UICONTROL Related Product Rule]**.
1. Create a disabled product that matches the rule (the rule can be kept simple by not adding any conditions under **[!UICONTROL Products to Match]** or **[!UICONTROL Products to Display]**).
1. Run the cron to execute the indexer backlog.
1. Edit the product and switch to the **[!UICONTROL Default Store View]** scope.
1. Enable the product **only** at the **[!UICONTROL Default Store View]** scope.
1. Run the cron to execute the indexer backlog.
1. Check the Storefront on a different product for the Related Products section, or query the `magento_targetrule_product` table to verify that the newly created product ID exists.

<u>Expected results</u>:

The product should be added to the **[!UICONTROL Related Product Rule]** (`magento_targetrule_product` table) by the indexer cron when created as disabled, and enabled only at the store view scope.

<u>Actual results</u>:

The indexer cron does not pick the product and add it to the **[!UICONTROL Related Product Rule]** (`magento_targetrule_product` table).
The product is added to the backlog of the `targetrule_product_rule_cl` table, but the indexer skips it during processing because it is enabled only at the store view scope.
