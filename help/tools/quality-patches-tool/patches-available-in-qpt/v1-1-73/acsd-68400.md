# ACSD-68400: The virtual gift card quantity was not accurately reflected in the inventory reservation table

The ACSD-68400 patch fixes the issue where the virtual gift card quantity was not accurately reflected in the inventory reservation table. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73 is installed. The patch ID is ACSD-68400. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.6-p12

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.7-p8

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where the virtual gift card quantity was not accurately reflected in the inventory reservation table.

<u>Steps to reproduce</u>:

1. Create clean 2.4-develop instance
1. Login to admin
1. Create a virtual Gift card
1. Edit the product and click on Advance inventory
1. Enable "Manage Stock" and set qty to 9999
1. Save the product
1. Go to Frontend and go to Category Gift card
1. Add a product with amount 25
1. Add a product with amount 50
1. Go to check and place and the order
1. Check the last entry in inventory_reservation table for the placed order. It would contain qty as 2
1. Again go to admin and open the placed order from Sales -> Order
1. Click Invoice and submit an invoice for the order
1. Again check the inventory_reservation it has reservation of only 1 qty instead of 2.

<u>Expected results</u>:

Reservation should be created for the no of qty which was order / invoiced, which is 2 in this case.

<u>Actual results</u>:

Reservation is only created for 1 qty instead of 2.
