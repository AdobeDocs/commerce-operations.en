# ACSD-68664: Scheduled update preview has errors on custom store domains

The ACSD-68664 patch fixes the issue where the scheduled update preview breaks when attempting to preview content for stores with custom domains. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 is installed. The patch ID is ACSD-68664. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.6-p10

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.6 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where the scheduled update preview breaks when attempting to preview content for stores with custom domains.

<u>Steps to reproduce</u>:

1. Create second website/store/storeview.
1. Assign different domain to each website.
1. Create a Content->Block.
1. Set scope - new website only.
1. Edit with Page Builder.
1. Add Row.
1. Add Elements->HTML Code.
1. Add this html code inside the HTML block:

```
<nav class=navigation-block>
  <div class=primary-nav>
    <div class=primary-nav-link>
      <a href=/ class=primary-nav-item>Home</a>
    </div>
  </div>
</nav>
```

1. Save and exit.
1. Include this new block to Home page:
    1. Open Content->Page
    1. Edit Home Page.
    1. Open Page builder:
    1. Add new Row
    1. Add Content->Block
    1. Select previously created block
    1. Save Home page
1. Create a new schedule update for Home page.
1. Preview scheduled update.
1. Click on "Home" link this custom navigation bar.

<u>Expected results</u>:

Custom store domains can be previewed as is and navigated within the preview iframe, without any CORS errors displayed in console.

<u>Actual results</u>:

Scheduled update preview breaks when attempting to preview content for stores with custom domains and CORS errors, because of redirect to another domain, are displayed in console (Block was created for non default website with different domain).
