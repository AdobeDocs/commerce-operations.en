# ACSD-69351: Gift card balances and expiration dates displayed across incorrect websites

The ACSD‑69351 patch fixes the issue where gift card balances and expiration dates display across incorrect websites without validating the assigned website scope. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 is installed. The patch ID is ACSD-69351. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where gift card balances and expiration dates are displayed not in accordance with the website scope assigned.

<u>Steps to reproduce</u>:

1. Create *Website A* and *Website B*, and then create a corresponding Store and Store View for each website.
1. Go to **[!UICONTROL Catalog]** > **[!UICONTROL Products]** > **[!UICONTROL Add Product]** > **[!UICONTROL Simple Product]**, set the [!UICONTROL Name], [!UICONTROL SKU], [!UICONTROL Price] (for example, *10.00*), and **[!UICONTROL Stock Status]** to *In Stock*. In the **[!UICONTROL Product in Websites]** section, select both *Website A* and *Website B*, and then **[!UICONTROL Save]** the product.
1. Navigate to **[!UICONTROL Marketing]** > **[!UICONTROL Gift Card Accounts]** > **[!UICONTROL Add Gift Card Account]**, set **[!UICONTROL Website]** to *Website A* only, configure the balance as *100.00*, set the **[!UICONTROL Status]** to *Enabled*, select a future expiration date, **[!UICONTROL Save]** the gift card account, and note the generated gift card code.
1. Open the storefront for *Website B* and add the product created earlier to the shopping cart.
1. On the cart page, locate the **[!UICONTROL Gift Card]** section, enter the gift card code assigned to *Website A*, and select **[!UICONTROL Check gift card status and balance]** without applying the gift card. The balance and expiration date display.
1. Alternatively, sign in to a customer account on *Website B*, go to **[!UICONTROL My Account]** > **[!UICONTROL Gift Card]**, enter the same gift card code, and check the balance. The balance and expiration date display.
1. Proceed to checkout on *Website B* and attempt to apply the same gift card code. The system correctly rejects the gift card due to website scope validation.

<u>Expected results</u>:

The balance must remain restricted to the assigned website, and the system must display the following error when applying the gift card: *Please correct the gift card account website: X*.

<u>Actual results</u>:

The system displays the gift card balance regardless of the assigned website restriction.
