# ACSD-69351: Gift card balances and expiration dates displayed across incorrect websites

The ACSD-69351 patch fixes the issue where gift cards balances and expiration dates are displayed not in accordance to the website scope assigned. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 is installed. The patch ID is ACSD-69351. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p8

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where gift cards balances and expiration dates are displayed not in accordance to the website scope assigned.

<u>Steps to reproduce</u>:

1. Create Website A and Website B, with corresponding Store, and Store View for each.
2. Create a simple product visible on all websites: Navigate to Catalog → Products → Add Product → Simple Product. Set Name, SKU, Price (e.g., 10.00), and Stock Status to In Stock. In the Product in Websites section, check both Website A and Website B. Save the product.
3. Create a gift card account restricted to Website A only: Navigate to Marketing → Gift Card Accounts → Add Gift Card Account. Set Website to "Website A" only. Set Balance to 100.00, Status to Enabled, and a future Expiration Date. Save and note the generated gift card code.
4. Open the Website B in storefront and add the product created in step 2 to cart.
5. On the cart page, locate the Gift Card section, enter the gift card code (assigned to Website A only), and click "Check gift card status and balance" (do not apply). Balance and expiration date are displayed.
6. Alternatively, log in to a customer account on Website B, navigate to My Account → Gift Card, and enter the same code to check balance. Balance and expiration date are displayed.
7. For comparison, attempt to actually apply the same gift card code at checkout on Website B. Gift card is correctly rejected — the scope validation works when applying, just not when checking balance.

<u>Expected results</u>:

Balance should be restricted by website and an error should display when applying gift card:
Please correct the gift card account website: X

<u>Actual results</u>:

Gift card balance is displayed regardless of website restriction.
