# ACSD-68341: Multiple X‑Magento‑Vary cookie updates occur on PDP load

The ACSD-68341 patch fixes the issue where X-Magento-Vary cookie sets several times during PDP loading, when multiple customer segments are created at the store. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 is installed. The patch ID is ACSD-68341. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.6-p10

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where X-Magento-Vary cookie sets several times during PDP loading, when multiple customer segments are created at the store.

<u>Steps to reproduce</u>:

1. Create a product and a category.
1. Create three customer segments by navigating to: Customers -> Segments -> Add Segment:
    1. Assigned to Website: ALL
       Apply to: Registered Customers
    1. Assigned to Website: ALL
        Apply to: Visitors and Registered Customers
    1. Assigned to Website: ALL
        Apply to: Registered Customers
1. Create a customer and log in.
1. Go to the created PLP or PDP.
1. Open dev tools in the browser, go to Application -> Cookies and search for "Vary".
1. Check the value and save it. Reload the page and observe that the Cookie value changes twice per page load on every subsequent page reload.

<u>Expected results</u>:

X-Magento-Vary cookie value is consistent.

<u>Actual results</u>:

X-Magento-Vary cookie value changes several times per page load.
