# ACSD-69020: Configurable products appear in Page Builder carousels when child products match filters

The ACSD-69020 patch fixes the issue where a configurable product is automatically included in PageBuilder product carousel listings if any of its child products meets the filtering conditions. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 is installed. The patch ID is ACSD-69020. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p7

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.7 - 2.4.7-p8

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where a configurable product is automatically included in PageBuilder product carousel listings if any of its child products meets the filtering conditions.

<u>Steps to reproduce</u>:

1. Create one content staging update for existing product, wait till update starts and run crons.
1. Create products with below sequence:
    1. configurable product "testconfig", with 3 options child products.
    1. 3 simple products with "testoption1", "testoption2", "testoption3" with each option 1,2,3.
    1. configurable product "testconfigtwo", with 3 options child product.
    1. make sure all simple products created above are assigned Visibility as 'catalog'.
1. Perform reindex or run crons.
1. Swap the options for each configurable product as below:
    1. Edit configurable product "testconfig", remove the "testconfig-option1" child product.
    1. Edit configurable product "testconfigtwo", remove the "testconfigtwo-option3" child product.
    1. Edit configurable product "testconfig", add the "testconfigtwo-option1" child product & Save.
    1. Same with configurable product "testconfigtwo", add the "testconfig-option3" child product & Save.
1. Perform reindex or run crons.
1. Go to admin -> Content -> Pages -> Edit any page.
1. Under content section using Edit with Page builder.
1. Add new Row.
1. add/edit content with Products options.
1. Set Select Products By = SKU.
1. Set the SKU as "testconfigtwo-option1" and save.
1. Observe configurable products in the carousel.

<u>Expected results</u>:

Only configurable products that have assigned child products not visible individually should be shown in Product carousel.

<u>Actual results</u>:

Configurable products that have "testconfigtwo-option 1" assigned to them are shown in Product carousel.
