# ACSD-53502: Add to Cart intermittently fails on iOS Safari due to New Relic script recursion

The ACSD-53502 patch fixes the issue where Add to Cart intermittently fails on the storefront in iOS Safari due to recursive calls to the New Relic monitoring script, causing page reloads. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 is installed. The patch ID is ACSD-53502. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.6.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.5-p12

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.5-p17

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where Add to Cart intermittently fails on the storefront in iOS Safari due to recursive calls to the New Relic monitoring script, causing page reloads.

<u>Steps to reproduce</u>:

1. Open Storefront on an iOS device.
1. Accept cookies if prompted.
1. Navigate to a product detail page.
1. Click Add to Cart.
1. Repeat the action multiple times, including in normal and incognito modes.

<u>Expected results</u>:

1. Clicking Add to Cart adds the selected product to the cart.
1. The mini cart updates without reloading the full page.

<u>Actual results</u>:

1. Clicking Add to Cart intermittently triggers a full page reload.
1. The product is not added to the cart.
1. The mini cart remains empty.
1. Cookie/session behavior may appear unstable.
