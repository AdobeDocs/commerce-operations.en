# ACP2E-4156: REST API shipping address validation ignores Admin attribute configuration

The ACP2E-4156 patch fixes the issue where shipping address validation in the REST API does not adhere to the attribute configuration defined in Admin. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 is installed. The patch ID is ACP2E-4156. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p4

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where shipping address validation in the REST API does not adhere to the attribute configuration defined in Admin.

<u>Steps to reproduce</u>:

1. Go to Admin > Stores > Attributes > Customer Address.
1. Open the attribute with an attribute code "firstname".
1. Under Attribute Properties, set:
    1. Input Validation: Length Only.
    1. Minimum Text Length: 1.
    1. Maximum Text Length: 20.
1. Go to the storefront and add the test product to the cart.
1. Send a REST API request to /rest/default/V1/guest-carts/<cartId>/shipping-information to add or update shipping information with a firstname value longer than 20 characters.

<u>Expected results</u>:

An error is returned that notifies about the use of a firstname value that does not respect the configured length limit.

<u>Actual results</u>:

Shopping cart shipping information is successfully updated with the firstname value sent in the REST API request.
