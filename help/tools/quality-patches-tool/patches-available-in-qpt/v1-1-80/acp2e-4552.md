# ACP2E-4552: GraphQL response doesn't return company status

The ACP2E-4552 patch fixes the issue where the company status is not returned in the GraphQL response. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 is installed. The patch ID is ACP2E-4552. Please note that this issue is scheduled to be fixed in Adobe Commerce B2B 1.5.2-p3.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) B2B 1.5.2-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) B2B >=1.5.0 <1.5.3

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where the company status is not returned in the GraphQL response.

<u>Prerequisites</u>:

1. [!DNL B2B] installed.

<u>Steps to reproduce</u>:

1. Set Admin > Stores > Settings > Configuration > General > [!DNL B2B] Features > Enable Company to Yes.
1. Log in to the Commerce Admin as an administrator.
1. Create three companies with the following statuses: Active, Rejected, and Pending Approval.
1. Go to Admin > Customers > All Customers, open a customer account, and from the Associate to Company list assign the customer to each of the created companies.
1. Send the following GraphQL request.

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies{
        items{
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>Expected results</u>:

The GraphQL response returns the correct status values (such as Active, Pending Approval, Rejected, and Blocked) for all companies associated with the customer, and the response can be filtered or sorted by company status.

<u>Actual results</u>:

The GraphQL response does not include the company status values for companies associated with the customer.
