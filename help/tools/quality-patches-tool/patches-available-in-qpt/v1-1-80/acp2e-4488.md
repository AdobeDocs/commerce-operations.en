# ACP2E-4488: Slow Admin product save/edit for large attribute sets

The ACP2E-4488 patch fixes the issue where saving or editing products in Admin is slow for products with large attribute sets. This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 is installed. The patch ID is ACP2E-4488. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.8-p3

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where saving or editing products in Admin is slow for products with large attribute sets.

<u>Steps to reproduce</u>:

1. Create at least 500 product attributes and assign them to an attribute set.
1. Go to the Admin panel.
1. Create a new product or edit an existing product using the attribute set with many attributes.
1. Click **Save**.

<u>Expected results</u>:

1. Product save or edit operation completes successfully without browser errors.
1. The Admin page remains responsive during the save process.

<u>Actual results</u>:

1. The Admin page becomes unresponsive when saving or editing the product.
1. A browser "Page Unresponsive" warning is displayed.
