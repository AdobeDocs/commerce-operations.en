# ACP2E-4808: Product weight displays without unit of measurement on Storefront

The ACP2E-4808 patch fixes the issue where the Weight attribute on the storefront product page displays only a raw numeric value in the "Additional Information" or "More Information" section without the configured unit of measurement (lbs or kgs). This patch is available when the [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 is installed. The patch ID is ACP2E-4808. Please note that this issue is scheduled to be fixed in Adobe Commerce 2.4.9.

## Affected products and versions

**The patch is created for Adobe Commerce version:**

* Adobe Commerce (all deployment methods) 2.4.7-p7

**Compatible with Adobe Commerce versions:**

* Adobe Commerce (all deployment methods) 2.4.5 - 2.4.9

>[!NOTE]
>
>The patch might become applicable to other versions with new [!DNL Quality Patches Tool] releases. To check if the patch is compatible with your Adobe Commerce version, update the `magento/quality-patches` package to the latest version and check the compatibility on the [[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use the patch ID as a search keyword to locate the patch.

## Issue

Issue where the Weight attribute on the storefront product page displays only a raw numeric value in the "Additional Information" or "More Information" section without the configured unit of measurement (lbs or kgs).

<u>Steps to reproduce</u>:

1. Go to Admin > Stores > Attributes > Product.
1. Open the Weight attribute.
1. Under Storefront Properties, set **Visible on Catalog Pages on Storefront** to **Yes**.
1. Go to Admin > Catalog > Products.
1. Open any product and make sure that there is a value entered in the Weight attribute.
1. Navigate to the product page on the storefront.
1. View the Additional Information or More Information section.
1. Observe the Weight row.

<u>Expected results</u>:

Weight displays with the configured unit, e.g., 2 lbs (or kgs depending on store configuration).

<u>Actual results</u>:

Weight displays as a number only (for example, 10.000000) with no unit displayed.
